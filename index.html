About This Guide

This guide is designed for intermediate users familiar with Linux command-line operations. It focuses on Debian/Ubuntu-based systems. Prerequisites include basic Linux knowledge, a valid TrackMania United Forever license, and administrative access to your server.

Quick Start

For experienced users, here's the fast track:

  1. Requirements: 1.5GHz CPU, 1GB RAM, 5GB disk, ports 2350/3450/5000 open.
  2. Download: Get TrackmaniaServer_2011-02-21.zip from community sources.
  3. Setup: Extract to /opt/tmserver, create user, set permissions.
  4. Configure: Edit dedicated.cfg with server details.
  5. Launch: Run in LAN mode first, then enable internet.
  6. Validate: Check logs for "Server is now listed on the master server."

Automated Installation Script

For advanced users, here's a basic bash script to automate the setup:

#!/bin/bash
# TrackMania Server Setup Script
SERVER_DIR="/opt/tmserver"
USER="tmuser"

# Create user and directories
sudo useradd -r -s /bin/false $USER
sudo mkdir -p $SERVER_DIR/Tracks $SERVER_DIR/MatchSettings
sudo chown -R $USER:$USER $SERVER_DIR

# Download and extract server
cd /tmp
wget https://example.com/TrackmaniaServer_2011-02-21.zip
sudo unzip TrackmaniaServer_2011-02-21.zip -d $SERVER_DIR
sudo chown -R $USER:$USER $SERVER_DIR

# Create basic config
cat > $SERVER_DIR/dedicated.cfg << EOF

    
        My Server
        16
    

EOF
sudo chmod 600 $SERVER_DIR/dedicated.cfg
sudo chown $USER:$USER $SERVER_DIR/dedicated.cfg

echo "Setup complete. Edit configs and run as $USER."

Note: Customize URLs, paths, and configs as needed. Run with sudo.

This guide provides a clear, well-ordered process to set up a TrackMania United dedicated server. Follow each step sequentially for a successful setup.

üèéÔ∏è Step 1: Prepare Your Environment

Verify system requirements, install dependencies, configure network settings, and create service accounts.

1.1 Verify System Requirements

  1. Check Operating System: Ensure your OS is compatible (Linux or Windows). 
    cat /etc/os-release
  2. Verify CPU and RAM: Check resources. 
    lscpu
free -h
  3. Confirm Disk Space: Ensure at least 5GB free. 
    df -h

1.2 Install Dependencies

  1. Update Package Manager: 
    sudo apt update
  2. Install Packages: 
    sudo apt install -y curl wget build-essential

    Note: Replace with actual packages needed, e.g., curl, wget for downloads.

  3. Verify Installation: 
    curl --version

1.3 Configure Network Settings

  1. Check Firewall Status: 
    sudo ufw status
  2. Allow Ports: 
    sudo ufw allow 8080/tcp
  3. Allow SSH (Important for remote access): 
    sudo ufw allow ssh
  4. Enable Firewall: 
    sudo ufw enable
  5. Verify Rules: 
    sudo ufw status verbose

1.4 Create Service Accounts

  1. Create User: 
    sudo adduser --system --group --no-create-home appuser
  2. Create Directory: 
    sudo mkdir /opt/myapp
  3. Assign Ownership: 
    sudo chown -R appuser:appuser /opt/myapp
  4. Verify Permissions: 
    ls -ld /opt/myapp

üèéÔ∏è Step 2: Download the Server Software

Locate, download, verify, and extract the official server package.

2.1 Locate and Download Package

  1. Find Sources: Use community forums or GitHub for TrackmaniaServer_2011-02-21.zip.
  2. Download: 
    cd /tmp
wget https://example.com/TrackmaniaServer_2011-02-21.zip
  3. Alternative: 
    curl -O https://example.com/TrackmaniaServer_2011-02-21.zip

2.2 Verify Integrity

  1. Generate Checksum: 
    sha256sum TrackmaniaServer_2011-02-21.zip
  2. Compare: Match with official checksum.

2.3 Extract Files

  1. Navigate to Directory: 
    cd /opt/myapp
  2. Extract: 
    sudo unzip /tmp/TrackmaniaServer_2011-02-21.zip
  3. Set Ownership: 
    sudo chown -R appuser:appuser /opt/myapp
  4. Clean Up: 
    rm /tmp/TrackmaniaServer_2011-02-21.zip

üèéÔ∏è Step 3: Install and Organize Server Files

Verify extraction, create subdirectories, and set permissions.

3.1 Verify File Placement

  1. List Files: 
    ls -lA

3.2 Create Subdirectories

  1. Create Tracks: 
    sudo mkdir /opt/myapp/Tracks
  2. Create MatchSettings: 
    sudo mkdir /opt/myapp/MatchSettings

3.3 Set Permissions

  1. Set Ownership: 
    sudo chown -R appuser:appuser /opt/myapp/Tracks
sudo chown -R appuser:appuser /opt/myapp/MatchSettings
  2. Apply Permissions: 
    sudo chmod -R 755 /opt/myapp
  3. Verify: 
    ls -lA /opt/myapp

üèéÔ∏è Step 4: Create and Configure Core Files

Copy templates, configure dedicated.cfg and matchsettings.txt.

4.1 Create Configuration Files

  1. Navigate: 
    cd /opt/myapp
  2. Copy dedicated.cfg: 
    sudo cp dedicated_template.cfg dedicated.cfg
  3. Copy matchsettings: 
    sudo cp matchsettings_template.txt MatchSettings/MyFirstRotation.txt
  4. Set Ownership: 
    sudo chown appuser:appuser dedicated.cfg MatchSettings/MyFirstRotation.txt
  5. Secure Config File: 
    sudo chmod 600 dedicated.cfg

4.2 Configure dedicated.cfg

  1. Edit File: 
    sudo nano dedicated.cfg
  2. Set Parameters:
    Parameter Value
    server_name "My Awesome Server"
    server_comment "New players welcome!"
    admin_pwd "SecretAdminPass123"
    server_port 2350
    server_p2p_port 3450
    masterserver_account "my_server_login"
    masterserver_password "MyMasterServerPass"

4.3 Configure Match Settings

  1. Edit File: 
    sudo nano MatchSettings/MyFirstRotation.txt
  2. Define Tracks and Modes: 
    // Example
Track_ID_01
{
  GameMode = "TimeAttack"
  LapLimit = 5
}

üèéÔ∏è Step 5: Launch Server for First Time

Create start script, test in LAN mode, verify logs.

5.1 Create Start Script

  1. Create File: 
    sudo nano start_server.sh
  2. Add Content: 
    #!/bin/bash
cd "$(dirname "$0")"
# Replace server_executable with the actual binary name (use ls -l to find it)
su - appuser -c "./TrackManiaServer.exe /dedicated_cfg=dedicated.cfg /lan"
  3. Make Executable: 
    sudo chmod +x start_server.sh
  4. Set Ownership: 
    sudo chown appuser:appuser start_server.sh

5.2 Test Launch

  1. Run Script: 
    sudo ./start_server.sh
  2. Check Logs: Look for successful messages.
  3. Stop Server: Press Ctrl+C.

üèéÔ∏è Step 6: Configure Port Forwarding and Firewall

Set up router forwarding and server firewall rules.

6.1 Router Port Forwarding

  1. Find Local IP: 
    hostname -I

    Note: The capital 'I' shows all IP addresses; use the local network IP (e.g., 192.168.x.x).

  2. Access Router: Log in to router admin panel.
  3. Add Rules: Forward UDP 2350, 3450; TCP 5000.

6.2 Server Firewall

  1. Check Status: 
    sudo ufw status verbose
  2. Add Rules: 
    sudo ufw allow 2350/udp comment 'Game Server Main Port'
sudo ufw allow 3450/udp comment 'Game Server P2P Port'
sudo ufw allow 5000/tcp comment 'Server List Query Port'
  3. Verify: 
    sudo ufw status verbose

üèéÔ∏è Step 7: Bring Server Online and Validate

Enable internet mode, launch publicly, verify registration.

7.1 Activate Internet Mode

  1. Edit Script: 
    sudo nano start_server.sh
  2. Remove /lan: Change to ./TrackManiaServer.exe /dedicated_cfg=dedicated.cfg

7.2 Launch Publicly

  1. Run Script: 
    sudo ./start_server.sh
  2. Monitor Logs: Check for authentication and registration success.

7.3 Validate

  1. Check Server List: Use game browser to find your server.
  2. Test Connection: Join from another device.

üèéÔ∏è Step 8: Remote Administration and Monitoring

Install and configure remote admin tools.

8.1 Install Tool

  1. Install PHP: 
    sudo apt install php php-cli php-xml php-mysql
  2. Download XASECO: 
    cd /tmp
wget https://example.com/XASECO_latest.zip
  3. Extract: 
    sudo mkdir /opt/xaseco
sudo unzip /tmp/XASECO_latest.zip -d /opt/xaseco
  4. Set Ownership: 
    sudo chown -R appuser:appuser /opt/xaseco

8.2 Configure Connection

  1. Edit Config: 
    sudo nano /opt/xaseco/config.xml
  2. Add Details: IP 127.0.0.1, Port 5000, Admin Password.

8.3 Launch and Test

  1. Create Script: 
    sudo nano /opt/xaseco/start_xaseco.sh

    Content:

    #!/bin/bash
cd "$(dirname "$0")"
su - appuser -c "php xaseco.php"
  2. Make Executable: 
    sudo chmod +x /opt/xaseco/start_xaseco.sh
  3. Launch: 
    sudo /opt/xaseco/start_xaseco.sh
  4. Test Commands: Use /admin kick, /admin next, etc.

Troubleshooting

Common issues and solutions:

  • Server not visible: Check port forwarding and firewall rules.
  • Authentication failed: Verify masterserver credentials.
  • Port binding error: Ensure ports are not in use by other services.
  • XML syntax error: Validate dedicated.cfg for typos.

Glossary

  • MatchSettings: Configuration file defining track rotation and game modes.
  • Masterserver: Central server for listing and connecting to game servers.
  • P2P Port: Port for peer-to-peer content sharing.
  • XML-RPC: Protocol for remote server administration.

If you encounter issues, refer to troubleshooting tips or community resources.
